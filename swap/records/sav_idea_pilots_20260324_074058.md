## Curated Interpretation

- The strongest easy idea is `support_nn`, but it is clearly task-selective rather than universally helpful.
- `support_nn` is very promising on multi-image BLINK tasks:
  - `blink_art_style`: `0.6750 -> 0.8750`
  - `blink_semantic_correspondence`: `0.1143 -> 0.3143`
- On more standard binary / FGVC classification tasks, the current SAV class mean is still stronger:
  - `naturalbench_vqa`: baseline wins
  - `eurosat`: baseline wins
  - `tinyimage`: essentially no change
- `class_bank4` is not a good default in this lightweight form. It helps on `blink_semantic_correspondence`, ties on `tinyimage`, but hurts on the other three pilots.
- The practical takeaway is:
  - keep the current mean-prototype SAV as the default
  - treat `support_nn` as a targeted variant for multi-image / correspondence-style tasks
  - do not prioritize the current `class_bank4` implementation
- If we continue from here, the highest-value next step is a hybrid rule:
  - use `support_nn` only on BLINK-style multi-image tasks
  - keep mean prototypes on standard single-image classification / VQA tasks
